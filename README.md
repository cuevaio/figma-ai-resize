# Figma AI Resize

AI-assisted frame resizing for Figma, built with Create Figma Plugin + a local Next.js backend.

The plugin analyzes a selected frame, asks a model for a layout plan, applies the plan in Figma, then runs additional refine passes to improve readability and visual balance.

## What this project does

- Select one frame in Figma.
- Choose a target aspect ratio (for example `1:1`, `16:9`, `9:16`).
- Run `Resize with AI` from the plugin UI.
- The plugin:
  1. Analyzes the source frame structure.
  2. Exports a screenshot reference.
  3. Calls the backend `/api/resize` endpoint for an initial plan.
  4. Applies the plan to a new frame.
  5. Runs refine passes through `/api/refine` and reapplies in place.

## Architecture

Two runtimes work together:

- `src/main.ts` (Figma main thread):
  - Reads selection and node data from the Figma document.
  - Exports frame screenshots.
  - Calls backend endpoints.
  - Applies layout transforms to nodes.
- `src/ui.ts` (plugin iframe UI):
  - Renders controls and status.
  - Sends typed messages to main thread.
- `web/` (Next.js backend):
  - Hosts AI proxy endpoints (`/api/resize`, `/api/refine`, `/api/screenshots/upload`).
  - Calls `openai/gpt-5.3-codex` through AI SDK.

## Setup

Install dependencies:

```bash
bun install
```

For backend env:

```bash
cp web/.env.example web/.env.local
```

Then set `AI_GATEWAY_API_KEY` in `web/.env.local`.

Start backend:

```bash
bun run dev
```

From `web/` directory.

Build plugin:


```bash

```

## Endpoint contract

The plugin calls `http://localhost:3000`.

### `GET /api/resize` and `GET /api/refine`

Health endpoints used before running each pass.

Expected shape:

```json
{
  "ok": true,
  "service": "hello-world-resize-backend",
  "route": "/api/resize",
  "timestamp": "2026-03-22T00:00:00.000Z"
}
```

### `POST /api/resize`

Generates the first adaptation layout.

Request shape:

```json
{
  "runId": "run-...",
  "input": {
    "analysis": {
      "sourceFrame": { "nodeId": "...", "name": "...", "width": 1200, "height": 800 },
      "targetFrame": { "width": 1080, "height": 1080, "presetId": "1:1" },
      "nodes": [],
      "screenshot": { "mimeType": "image/png", "url": "https://..." }
    }
  }
}
```

Response shape:

```json
{
  "ok": true,
  "data": {
    "layoutVersion": "2",
    "targetFrame": { "width": 1080, "height": 1080 },
    "nodes": [
      {
        "path": "root.0.1",
        "x": 64,
        "y": 120,
        "width": 520,
        "height": 72,
        "fontSize": 48
      }
    ]
  },
  "meta": {
    "requestId": "req_...",
    "model": "openai/gpt-5.3-codex",
    "durationMs": 1234
  }
}
```

### `POST /api/refine`

Refines a frame that was already adapted.

It receives:

- `input.analysis`: analysis of current adapted frame.
- `input.referenceScreenshot` (optional): screenshot of original source frame for style anchoring.

Response schema is identical to `/api/resize` (`layoutVersion: "2"`).

### `POST /api/screenshots/upload`

Uploads base64 PNG screenshots to blob storage so model requests use URLs instead of large inline payloads.

## Frame analysis schema

Before calling AI, the plugin creates a structured analysis payload per node (`src/adaptation-types.ts`).

Each analyzed node includes:

- Identity: `path`, `nodeId`, `name`, `type`, `parentPath`, `zIndex`
- Visibility: `isVisible`
- Semantics: `classification`
- Geometry:
  - `absolute`: `{ x, y, width, height }` in page coordinates
  - `normalized`: `{ left, top, width, height, centerX, centerY }` relative to source frame
- Text metadata (for text nodes): character count, line estimate, and font-size range

Classification values:

- `BACKGROUND`
- `TEXT`
- `ICON`
- `LOGO`
- `RAIL`
- `MENU`
- `CONTENT`
- `DECORATION`

Layout plan schema returned by model:

- `layoutVersion: "2"`
- `targetFrame: { width, height }`
- `nodes[]`: each node has `path`, `x`, `y`, `width`, `height`, `fontSize`
  - `fontSize` is a number for text nodes, `null` for non-text nodes

All model output is validated and normalized before application (`src/adaptation-plan-schema.ts`).

## Figma API basics used here

Core API concepts in this plugin:

- Selection:
  - Uses `figma.currentPage.selection` and requires exactly one `FRAME` node.
- Geometry:
  - Uses `absoluteBoundingBox` and utility fallback `computeBoundingBox(...)`.
- Rendering references:
  - Uses `frame.exportAsync({ format: 'PNG', constraint: { type: 'SCALE', value } })`.
- Document mutation:
  - Resizes/repositions nodes and updates text settings in main thread.
- UX continuity:
  - Updates selection and calls `figma.viewport.scrollAndZoomIntoView(...)` after apply.
- User feedback:
  - Sends progress to UI via typed messages and shows `figma.notify(...)` for key outcomes.

## Model behavior goals

Prompts for resize and refine enforce:

- Keep important content in frame.
- Preserve text hierarchy and readability.
- Prevent new overlaps unless already present in source overlap policy.
- Preserve non-text aspect ratios.
- Reduce font sizes when text boxes are reduced.

## Next steps

1. Improve prompts with stronger guardrails for edge cases (dense text blocks, tiny mobile targets, asymmetrical hero layouts).
2. Benchmark multiple models across fixed frame datasets (quality + latency + token usage), then choose default per pass (`resize` vs `refine`).
3. Add an interactive `Refine` button in the plugin UI for user-driven additional passes after initial output.
4. Improve UX around refinement:
   - show pass-by-pass diff preview,
   - offer quick rollback to prior pass,
   - surface concise warnings with actionable suggestions.
