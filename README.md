# Hello World

## Development guide

*This plugin is built with [Create Figma Plugin](https://yuanqing.github.io/create-figma-plugin/).*

### Pre-requisites

- [Node.js](https://nodejs.org) – v22
- [Figma desktop app](https://figma.com/downloads/)

### Build the plugin

To build the plugin:

```
$ bun run build
```

This will generate a [`manifest.json`](https://figma.com/plugin-docs/manifest/) file and a `build/` directory containing the JavaScript bundle(s) for the plugin.

To watch for code changes and rebuild the plugin automatically:

```
$ bun run watch
```

### Install the plugin

1. In the Figma desktop app, open a Figma document.
2. Search for and run `Import plugin from manifest…` via the Quick Actions search bar.
3. Select the `manifest.json` file that was generated by the `build` script.

### Debugging

Use `console.log` statements to inspect values in your code.

To open the developer console, search for and run `Show/Hide Console` via the Quick Actions search bar.

## AI adaptation workflow

The plugin now supports an AI-assisted adaptation flow in addition to basic resize.

### What it does

- Analyze the selected frame structure and relative layout.
- Optionally export a screenshot reference of the frame.
- Ask `openai/gpt-5.3-codex` through a local backend proxy endpoint in `web/`.
- Run up to 3 adaptation passes (early-stopping when quality is good).
- Apply deterministic transforms in Figma (background first, then content), with text-specific scaling that prioritizes font size changes over box squeezing.

Note: The plugin calls `http://localhost:3000/api/resize`. Run the `web` app locally for AI adaptation.

### How to use

1. Select a single frame.
2. Choose the target aspect ratio.
3. Start backend proxy:
   - In `web/`, copy `.env.example` to `.env.local`
   - Set `AI_GATEWAY_API_KEY`
   - Run `bun run dev`
4. Click `Adapt with AI` in the plugin.

If AI planning fails or returns invalid data, the plugin applies a local fallback adaptation strategy.
